##[230.Kth Smallest Element in a BST](https://leetcode.com/problems/kth-smallest-element-in-a-bst/description/ "230.Kth Smallest Element in a BST")

####Similiar Topics:

<div class="question-description__3U1T" style="padding-top: 10px;"><div><p>Given a binary search tree, write a function <code>kthSmallest</code> to find the <b>k</b>th smallest element in it.</p>

<p><b>Note: </b><br/>
You may assume k is always valid, 1 &#8804; k &#8804; BST's total elements.</p>

<p><strong>Example 1:</strong></p>

<pre><strong>Input:</strong> root = [3,1,4,null,2], k = 1
   3
  / \
 1   4
  \
&#160;  2
<strong>Output:</strong> 1</pre>

<p><strong>Example 2:</strong></p>

<pre><strong>Input:</strong> root = [5,3,6,2,4,null,null,1], k = 3
       5
      / \
     3   6
    / \
   2   4
  /
 1
<strong>Output:</strong> 3
</pre>

<p><b>Follow up:</b><br/>
What if the BST is modified (insert/delete operations) often and you need to find the kth smallest frequently? How would you optimize the kthSmallest routine?</p>
